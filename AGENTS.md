# Nano Edit Agent Instructions

## Repo Instructions

- Demo surfaces should keep unnecessary explanation and decoration extremely restrained.
- Prefer document-like guidance inside editable content over surrounding tutorial chrome.
- Treat Nano Edit as an embeddable editor package; the local app is a demo host.

## Agent skills

### Issue tracker

Issues are tracked in GitHub Issues for `developer-1px/nano-edit`. See `docs/agents/issue-tracker.md`.

### Triage labels

Use the repo's GitHub triage labels for category and state roles. See `docs/agents/triage-labels.md`.

### Domain docs

This is a single-context repo. Read `CONTEXT.md` before changing domain concepts, and check `docs/adr/` if present. See `docs/agents/domain.md`.
